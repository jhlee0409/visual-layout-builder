# Dev Log Directory

이 디렉토리는 개발 작업 로그와 컨텍스트를 저장하는 공간입니다.

## 목적

- **컨텍스트 유지**: 장시간 작업 시 컨텍스트 손실 방지
- **작업 이력 추적**: Phase별 진행 상황 및 결정사항 기록
- **지식 공유**: 다음 작업자를 위한 힌트 및 주의사항
- **디버깅 지원**: 문제 발생 시 이전 결정사항 참조

## 파일 명명 규칙

```
YYYY-MM-DD-[작업명].md

예시:
- 2024-01-15-add-grid-item-positioning.md
- 2024-01-20-refactor-validation-logic.md
- 2024-02-05-fix-breakpoint-inheritance-bug.md
```

## 사용 가이드

### 작업 시작 시

1. 새 파일 생성: `YYYY-MM-DD-[작업명].md`
2. 템플릿 복사
3. 작업 목표 작성
4. Plan 및 Task 분해

### 작업 중

1. 각 Task 완료 시 체크박스 업데이트
2. 30분마다 Checkpoint 작성
3. Phase 완료 시 Phase Summary 작성
4. 중요한 발견사항은 즉시 기록

### 작업 완료 시

1. 최종 검증 체크리스트 완료
2. 최종 요약 작성
3. 다음 작업자를 위한 힌트 작성
4. CLAUDE.md 업데이트 (필요 시)

## 예시

실제 작업 예시는 이 디렉토리의 다른 파일들을 참고하세요:

- `2024-01-15-schema-v2-migration.md`: Schema V1 → V2 마이그레이션
- `2024-01-20-canvas-grid-system.md`: Canvas Grid 시스템 구현
- `2024-02-01-validation-enhancement.md`: Validation 로직 강화

## 보관 정책

- **활성 로그**: 최근 30일 이내 작업
- **보관 로그**: 30일 이상 경과 (참고용)
- **삭제 대상**: 6개월 이상 경과 + 더 이상 관련 없는 작업

## 주의사항

- 민감한 정보 (API 키, 비밀번호 등)는 절대 기록하지 마세요
- 코드는 가능한 한 짧게 발췌하고, 전체 코드는 파일 경로로 참조하세요
- 이모지 사용: ✅ (완료), 🔄 (진행중), ❌ (실패/차단)
